# Clases abstractas
* No instanciable pero puede tener constructores y atributos
* metodos vacios, solo tiene cabecera 
* Hijas, nietas u otros descendientes tendran que poner el codigo de esos metodos (@Override)
* Etiqueta abstract antes de clase
* 0% 50% 100% modelos abstractos
# Interfaces
* definicion de comportamiento
* Metodos vacios, todos.
* En versiones modernas, metodos con codigo. (etiqueta default)
* puede tener atributos constantes, static final, no hace falta poner ninguna etiqueta en ellos
* se relaciona con otra clase con implements

##### Recuerda que final, evita que se reescriba
